핵심 결론: 블리자드 레이드프레임 디버프 목록은 건드리지 않는다

minicc는 CompactUnitFrame의 기존 buffFrames/debuffFrames를 조작하지 않는다. 모든 아이콘 크기 조작은 minicc가 자체 생성한 IconSlotContainer 프레임 내에서만 이루어진다.

CC 걸렸을 때 "디버프가 크게 보이는" 현상은 — minicc의 CrowdControlModule이 각 CompactUnitFrame 위에 별도 아이콘 컨테이너(기본 32px)를 오버레이하기 때문이다. 블리자드 원본 아이콘 크기 변경이 아님.

주의: FriendlyIndicatorModule만 CompactUnitFrame 훅 사용

hooksecurefunc("CompactUnitFrame_SetUnit", ...) 및 hooksecurefunc("CompactUnitFrame_UpdateVisible", ...)를 사용하지만, 이는 유닛 변경/표시 감지용이며 아이콘 크기와 무관하다.

핵심 엔진 크기 조작

minicc의 모든 아이콘 표시를 담당하는 중앙 엔진. 슬롯 풀, 레이어 시스템, 쿨다운, 글로우, Masque 스킨 통합.

  • M:New(parent, count, size, spacing, groupName)
    새 컨테이너 생성. size는 px 단위 고정값.
  • M:SetIconSize(newSize)
    모든 슬롯·레이어의 SetSize() 일괄 호출 + 폰트 크기 자동 조정
  • M:SetSlot(index, options)
    슬롯에 텍스처·쿨다운·색상·글로우 적용
  • M:Layout()
    슬롯 위치 재계산 (시그니처 비교로 중복 스킵)
function M:SetIconSize(newSize)
  if self.Size == newSize then return end
  self.Size = newSize
  for i = 1, self.Count do
    local slot = self.Slots[i]
    slot.Frame:SetSize(self.Size, self.Size)  -- 자체 프레임만!
    fontUtil:UpdateCooldownFontSize(layer.Cooldown, self.Size)
  end
end

폰트 크기 계산

아이콘 크기에 비례해 쿨다운 숫자 폰트 크기를 자동 계산. SetIconSize() 호출 시 항상 연동.

function M:UpdateCooldownFontSize(cd, iconSize, coeff, fontScale)
  local coeff = coeff or 0.4
  local fontSize = iconSize * coeff * (fontScale or 1.0)
  -- fontSize = iconSize × 0.4 × fontScale
end

37개 파일 전체 분석 결과 요약

블리자드 debuffFrames 조작 없음

minicc의 어떤 코드도 frame.buffFrames, frame.debuffFrames를 직접 건드리지 않는다. CompactUnitFrame_UpdateAuras 훅도 없다. 블리자드 원본 디버프 아이콘 크기는 변하지 않는다.

CC 크게 보이는 이유

CrowdControlModule이 각 CompactUnitFrame 위에 별도 IconSlotContainer(기본 32px)를 앵커링한다. 이 오버레이 아이콘이 블리자드의 작은 디버프 아이콘보다 커서 "CC 디버프가 크게 보이는" 것처럼 느껴지는 것이다.

FriendlyIndicator CUF 훅

minicc에서 유일하게 hooksecurefunc("CompactUnitFrame_SetUnit", ...)를 사용하지만, 이는 유닛 변경 감지 목적이다. 아이콘 크기나 debuffFrame과 무관하다.
